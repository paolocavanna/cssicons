Unicode symbols in css: a list

Using icons in visual communication is an established pattern to convey a message in a rapid way or to support a verbal one.

Icons are widely used in web pages: since some times ago, web authors relied on images to embed them in their pages; nowadays, many deisgners have created stylish icon sets made by fonts, thanks to the widespread support of @font-face css property.

In some cases, you may not have to look for over complex solution, but, instead, go for simplicity. For example, it's a common design pattern to separate breadcrumb navigation items with "\". Giving a structure like this: 

			
<ul>
	<li><a href="#">Home</a></li>
	<li><a href="#">Products</a></li>
	<li>This product</li>
</ul>

To do so, you have at least three solution:

  • an empty element before or after the list items, with the "\" (slash) as text node;
  • the "\" (slash) as backgroud image of the list items;
  • a pseudo-element1 ::before or ::after the list items, with its content set to the right unicode character2.

The first is a non-semantic solution, so it should be deprecated. The second has two downsides: the browser needs to download an extra image and, for retina displays, you should provide a 2x resolution image. The third solution is clean, fast and works in all browsers (except for IE7 down): no extra image, no resolution issue; moreover, you can style your pseudo-element via css typographic properties.

Since I've found myself exploiting that third solution many times, I thought it'd be useful to prepare a set of css classes ready to be dropped in any html element. Sharing this simple work is the second step.

For a full reference about unicode characters, visit fileformat.info

Notes

  1. The double colons syntax for pseudo-elements (::before, ::after) is part of CSS3. If you want to support older browsers like IE8, you need to use the old syntax (:before, :after). Further details here;
  2. your css document must be encoded in UTF-8 to properly render unicode characters. To do so, use the meta tag <meta charset="utf-8" /> in the <head> of your document or use a media rule in css @charset "UTF-8";. Remember that css inherits charset from html document, so css @charset is useless if html is correctly set to UTF-8.

Here you have the css ready to be copied and pasted. I've used the :after pseudo-element, but you could use :before depending on which fits best for you. 

				
				/* === ARROWS === */
				.icon-arrow-rgt:after { content: "\2192" }
				.icon-arrow-lft:after { content: "\2190" }
				.icon-arrow-up:after { content: "\2191" }
				.icon-arrow-dwn:after { content: "\2193" }

				/* === TRIANGLES === */
				.icon-triangles-rgt-blck:after { content: "\25b6" }
				.icon-triangles-rgt-blck-small:after { content: "\25b8" }
				.icon-triangles-dbl-rgt-blck:after { content: "\23E9" }
				.icon-triangles-rgt-wht:after { content: "\25b7" }

				.icon-triangles-lft-blck:after { content: "\25c0" }
				.icon-triangles-lft-blck-small:after { content: "\25c2" }
				.icon-triangles-dbl-lft-blck:after { content: "\23ea" }
				.icon-triangles-lft-wht:after { content: "\25c1" }

				.icon-triangles-dwn-blck:after { content: "\25bc" }
				.icon-triangles-up-blck:after { content: "\25b2" }
				.icon-triangles-up-wht:after { content: "\25b3" }
				.icon-triangles-dwn-wht:after { content: "\25bd" }

				/* === MISCELLANEOUS === */
				.icon-misc-menu:after { content: "\2261" }
				.icon-misc-close:after { content: "\2573" }
				.icon-misc-checked:after { content: "\2713" }
				.icon-misc-pipe:after { content: "\007C" }
				.icon-misc-emdash:after { content: "\2014" }
				.icon-misc-backslash:after { content: "\005C" }
				.icon-misc-minus:after { content: "\2212" }
				.icon-misc-plus:after { content: "\002b" }

				/* === CURRENCY SYMBOLS === */
				.icon-curr-euro:after { content: "\20AC" }
				.icon-curr-pound:after { content: "\00a3" }
				.icon-curr-usdollar:after { content: "\0024" }
